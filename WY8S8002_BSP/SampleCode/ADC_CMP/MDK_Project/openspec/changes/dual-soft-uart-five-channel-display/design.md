## Context

现有工程以 16 MHz HRC 系统时钟运行，P01 仍为 UART0 RX，P06 为单路 9600 软件 RX，并在每一位采样时重装 Timer2。硬件 UART0 无法满足 115200；两路上游帧已确认不重叠。P03/P05 分别提供有线电压/电流，TM1622 已支持五路显示，P17 与 Timer3 未被现有业务占用。

## Goals / Non-Goals

**Goals:**

- 在 P01/P06 接收不重叠的 115200、8N1 数据流，可靠发布 C1-C4 数据。
- 以模块接口隔离 GPIO、定时器、协议、ADC 和显示，并由 `g_app` 作为业务层入口。
- 提供 P17 背光按键和五路电压、电流、功率显示。

**Non-Goals:**

- 不实现 UART 发送、同时重叠报文接收、校验和、动态内存或外部通信协议。
- 不改变 TM1622 的段码布局和 P02/P04 ADC 既有采样能力。

## Decisions

- **P01/P06 共用 GPIO ISR，Timer2 独占串口位采样。** GPIO ISR 仅识别下降沿和锁存端口；因帧不重叠，活跃帧期间另一端口请求被计数丢弃。相对于双定时器方案，这避免 115200 下并发中断负担。
- **Timer2 采用自动重载。** 起始边沿后装载半位延时以确认起始位；确认后写入完整位周期重载值、开启溢出自动重载，连续在位中心采样数据与停止位，不在每个数据位软件重装计数。
- **协议层拥有重同步和帧校验。** 每个端口各自维护 11 字节帧状态；只接受 `A5` 起始、`5A` 结束和 1/2/3 连接位，失败时以当前字节重新寻找 `A5`。
- **业务层拥有五路数据和显示转换。** 业务层从协议和 ADC 读取数据，使用 32 位中间值计算 `mV*mA/1000` 得到 mW，再转换为 TM1622 的 0.1A/0.1V/0.1W 输入单位。
- **Timer3 提供 1 ms 软件时基。** UART0 移除后 Timer3 用于 P17 的 20 ms 非阻塞消抖和显示刷新节拍；按键 ISR 不调用延时。

## Risks / Trade-offs

- **[Risk] 两路帧重叠会丢失后到达的帧。** → 上游保证不重叠，驱动提供忙丢失计数以便板测确认。
- **[Risk] 内部 HRC 偏差影响 115200 裕量。** → 按当前系统时钟计算重载值，并以逻辑分析仪核验位时间和完整帧接收。
- **[Risk] 不规范帧导致旧数据保留。** → 仅完整有效帧更新测量数据，维护无效帧计数。
- **[Risk] 功率或显示值超过段码能力。** → 用 32 位计算功率，并在转换到显示输入时饱和至显示字段上限。

## Migration Plan

1. 新增变更规格和任务，修改板级资源所有权。
2. 实现双路软件 RX、协议解析、按键时基和业务层映射。
3. 从 Keil 活跃构建移除 UART0 驱动，加入新模块并构建。
4. 先用示波器确认 Timer2 位周期，再分别注入两路标准帧并验证显示与背光。
